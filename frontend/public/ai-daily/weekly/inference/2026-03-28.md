# ⚡ AI推理加速周报 - 03.23-03.30

## 🏆 本周Top进展

### 1. vLLM v0.18.0 大版本发布（3月20日发布，本周活跃迭代）
本周最重磅的推理框架更新。445个commit、213位贡献者（61位新贡献者）：
- **gRPC Serving 支持**：新增 `--grpc` flag，除 HTTP/REST 外提供高性能 RPC 服务
- **NGram GPU 推测解码**：NGram 推测解码全面移至 GPU 并兼容异步调度器，显著降低 spec decode 开销
- **KV Cache 智能卸载**：仅存储高频复用 block，新增 FlexKV 后端，支持多 KV group 卸载
- **弹性专家并行 Milestone 2**：NIXL-EP 集成实现 MoE 动态 GPU 扩缩容
- **FlashInfer 0.6.6** 升级，含大量性能和正确性改进
- **Eagle3 推测解码**：支持 Qwen3.5 和 Kimi K2.5 MLA
- Ray 不再是默认依赖

### 2. SGLang v0.5.10rc0 发布（3月28日）
本周另一个重大发布，多项核心优化：
- **Piecewise CUDA Graph 默认启用**：减少复杂控制流模型的内存开销，提升吞吐
- **弹性 EP 容错**：集成 Elastic NIXL-EP，DeepSeek MoE 部署可 GPU 故障自动重新分配 expert 权重，无需重启
- **HiSparse 稀疏注意力**：集成 HiSparse 稀疏注意力后端，长上下文推理高效降低计算量
- **FlashInfer MXFP8 Kernel**：GEMM 和 MoE 支持 microscaling FP8 混合精度推理
- **Transformers 5.3.0 升级**：GLM-5 可直接在主线分支运行
- **DeepSeek V3.2 / GLM-5 优化**：Fused Triton KV cache prefill kernel、NSA fuse store indexer
- **FlashAttention 4 官方库支持**：最新注意力优化 + Blackwell GPU 支持
- **原生 MLX 后端**：Apple Silicon 原生推理
- **sglang-kernel 0.4.0** 发布

### 3. TensorRT-LLM 新预发布版（3月24日）
NVIDIA 推理框架密集更新：
- **Qwen3.5 文本模型 PyTorch 后端 BF16/FP8 支持**
- **DeepSeek-V3.2 NVFP4** indexer GEMM 和 routing kernel 优化
- **Trtllm-Gen attention 后端** 支持 KV cache 和推测解码
- **GLM-5 支持** + DSA MTP 问题修复
- 新增**能量指标**到 trtllm-serve
- Fused AllReduce + RMSNorm kernel
- **FlashMLA 性能优化**：新 kernel fusion + cached tile-scheduler metadata
- SM103 context-attention kernel 修复（Blackwell 准备）

### 4. Ollama v0.18.0 发布
面向终端用户的重要更新：
- **Nemotron-3-Super 122B** 支持（需 96GB+ VRAM 本地运行），PinchBench 最高分
- **Kimi-K2.5 速度提升 2x**，tool calling 准确率提升
- **ROCm 7 升级**，需更新 AMD 驱动
- 非交互式任务支持（`--yes` flag）
- MiniMax-M2.5 和 Qwen3.5 云推理 TTFT 大幅降低（最高 10x/2x）

### 5. vLLM Semantic Router v0.2 Athena（3月10日发布，本周持续影响）
语义路由的系统级升级：
- **mmBERT-Embed-32K** 多语言长上下文骨干：32K context、1800+ 语言、307M 参数
- **Multi-Modal Embed Small**：120M 参数，文本/图像/音频统一 384d 空间
- **ClawOS 实验层**：Semantic Router 编排多 OpenClaw 系统的路由/记忆/安全管理

## 📈 趋势洞察

### 1. MoE 推理进入弹性时代
本周最显著的趋势：**vLLM 和 SGLang 几乎同时落地 Elastic NIXL-EP**。这意味着 MoE 模型（DeepSeek-V3.2 等）的推理服务终于有了生产级容错能力——单个 GPU 故障不再意味着全服务重启。随着 MoE 成为大模型主流架构，弹性专家并行将成为推理框架的必备特性。

### 2. 推测解码全面 GPU 化
vLLM 的 NGram spec decode 全面 GPU 化、Eagle3 支持扩展到 Qwen3.5 和 Kimi K2.5 MLA、TRT-LLM 在 Trtllm-Gen attention 后端启用推测解码——推测解码正在从"可选优化"变为"默认开启"的标配。

### 3. FP8 → MXFP8：混合精度进入 microscaling 时代
SGLang 集成 FlashInfer MXFP8 kernel，vLLM 推出 FP8 LoRA dense kernel。MXFP8 通过 microscaling 在更细粒度上管理精度，相比传统 FP8 在 RL 和通用推理场景有更好的精度-性能权衡。这是量化精度提升的下一步。

### 4. 多模态推理的框架支持加速
vLLM v0.18 增加了音频转录、视频裁剪、ColPali late-interaction 检索等多模态推理能力，SGLang 扩展了 diffusion 模型支持（LTX-2、Hunyuan3D-2）。推理框架正从纯文本加速器演变为多模态推理平台。

### 5. 端侧推理持续下沉
llama.cpp 对 CUDA 13、ROCm 7.2、OpenVINO 2026.0、昇腾 910b、Apple MLX 的全面覆盖表明：边缘和端侧推理的硬件生态正在快速成熟。Ollama 的 ROCm 7 升级和 MLX 后端的出现，让非数据中心场景的推理部署门槛进一步降低。
