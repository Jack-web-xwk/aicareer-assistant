# 🧬 AI微调日报 - 2026-03-29

## 🔥 要闻

### 1. 🚀 Unsloth 推出推理加速更新，速度提升20-30%
Unsloth 3月27日发布更新，修复了此前 tool-calling 和 repeat penalty 导致推理速度下降的问题，推理 tokens/s 现已接近 llama.cpp 原生水平。同时新增自动识别 LM Studio / HuggingFace 已下载模型、训练历史查看等功能。
🔗 [Unsloth GitHub Releases](https://github.com/unslothai/unsloth/releases)

### 2. ⚡ Axolotl v0.15.0：ScatterMoE LoRA + MoE专家量化，QLoRA显存狂降5倍
Axolotl v0.15.0 新增 ScatterMoE LoRA 支持和 SonicMoE MoE kernel，升级至 Torch 2.10.0。重磅特性是 MoE Expert 量化——以 GLM-4.7-Flash QLoRA 为例，显存从 ~127 GiB 降至 ~23 GiB，开启方式：`quantize_moe_experts: true`。同时新增 Qwen3.5 / GLM / Step3.5 模型支持。
🔗 [Axolotl GitHub Releases](https://github.com/axolotl-ai-cloud/axolotl/releases)

### 3. 🔄 vLLM v0.18.0：gRPC服务 + KV缓存卸载 + 弹性专家并行
vLLM v0.18.0 带来 445 commits / 213 贡献者。核心更新：gRPC 服务支持、NGram GPU 推测解码、KV Cache 智能 CPU 卸载（FlexKV 后端）、NIXL-EP 弹性专家并行动态 GPU 扩缩容、FlashInfer 0.6.6 更新。对微调后的 MoE 模型部署尤为关键。
🔗 [vLLM GitHub Releases](https://github.com/vllm-project/vllm/releases)

### 4. 🏭 LlamaFactory 扩展模型矩阵：Qwen3.5 / MiMo / MiniCPM4 / Mistral-Small-3.1
LlamaFactory 持续跟进新模型，近期新增 Qwen3.5、MiMo/MiMo-VL、MiniCPM4、Mistral-Small-3.1、Qwen2.5-Omni、Llama 4 / Gemma 3 多模态等模型支持，并新增 SGLang 推理引擎和官方 GPU Docker 镜像。
🔗 [LlamaFactory GitHub Releases](https://github.com/hiyouga/LlamaFactory/releases)

### 5. 📄 arXiv 前沿：SFT-DPO 交互研究 + 联邦个性化DPO
两篇值得关注的新论文：
- **SFT-DPO Interaction in Small Language Models** — 实证研究 SFT 和 DPO 在小模型上的交互与参数化策略，为小模型后训练提供实践指导
- **FedPDPO** — 提出联邦个性化 DPO，在去中心化、非IID偏好数据下实现 LLM 对齐
🔗 [arXiv: SFT-DPO](https://arxiv.org/search/?query=fine-tuning+LoRA+DPO&searchtype=all)
