## 🔧 技术速报

### 量化与精度

- **NVFP4 on Blackwell**：vLLM 修复了 NVFP4 量化在 DGX Spark（GB200）和 RTX 50 系列上的兼容性问题，Blackwell 消费级推理可用性提升
- **量化权重加载修复**：vLLM 修复 dummy weight loading bug，影响 GPTQ/AWQ 等量化模型的加载流程
- **Mamba 随机舍入**：NVIDIA 为 vLLM 的 Mamba 模型添加 stochastic rounding 支持，提升 FP8/低精度推理下的数值稳定性

### KV Cache 与注意力

- **FlashAttentionDiffKV 解耦**：vLLM 将 KV-Cache 更新从 FlashAttention 内核中提取为独立 Triton 操作，为 DiffKV（差分 KV）优化扫清架构障碍
- **DCP CUDA Graph 修复**：Decode Context Parallelism 的 CUDA Graph 捕获 bug 被修复，多 GPU 推理稳定性提升
- **HMA 边界修复**：Hybrid Memory Attention 在 `blk_size=64, tp=4` 时的整除边界问题被修复

### 推测解码

- **DFlash（vLLM）**：全新推测解码架构，使用双向注意力 + 独立 KV Cache 写入 + 异步调度，相比 P-EAGLE 有架构级改进
- **EAGLE Norm-before-FC（TRT-LLM）**：推测解码草稿模型在全连接层前加归一化，修复 shape mismatch 场景

### 硬件与多平台

- **Adreno GPU 量化**：llama.cpp 为高通 Adreno GPU 添加 Q4_K 量化内核，移动端 LLM 推理再进一步
- **NPU Ring Attention**：SGLang 在华为 NPU 上支持 Ring Attention + Flash Attention，扩散模型场景可用
- **NPU 并行解码**：SGLang 支持在 NPU 上对 Qwen-Image 进行并行解码
- **AMD 性能回归修复**：SGLang 修复 AMD 上长上下文推理的性能回归问题

## 📈 今日洞察

今天的关键词是 **"推测解码百花齐放"**。vLLM 的 DFlash 是本日最值得关注的动态——它不是简单的工程优化，而是对推测解码架构的重新思考：通过将 context states 直接写入 KV Cache 并利用异步调度，避免了传统 draft 模型的串行瓶颈。与此同时，TensorRT-LLM 的 EAGLE 也在做精细化改进（Norm-before-FC），两大框架在推测解码赛道上的竞争愈发激烈。另一边，NVIDIA 的工程师正在全面推进 Blackwell 适配（NVFP4 + MLA + DCP），vLLM 已经成为 NVIDIA GPU 推理生态的核心入口。SGLang 则继续在 AMD/NPU 异构路线上深耕，多硬件推理格局正在加速分化。
