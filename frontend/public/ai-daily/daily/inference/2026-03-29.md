# ⚡ AI推理加速日报 - 2026-03-29

## 🔥 要闻

**1. SGLang v0.5.9 发布：集成TRT-LLM NSA内核，DeepSeek V3.2推理提速3-5倍**
SGLang v0.5.9 集成了 TensorRT-LLM 的 Native Sparse Attention (DSA) 内核，在 Blackwell 平台上为 DeepSeek V3.2 带来 3x-5x 的 prefill/decode 性能提升。同时新增 FP4 注意力 (FA4)、FlashInfer All-to-All MoE Dispatcher、LoRA 权重加载与计算重叠（TTFT 降低 ~78%）等多项重大优化。
🔗 [SGLang v0.5.9 Release](https://github.com/sgl-project/sglang/releases/tag/v0.5.9)

**2. vLLM v0.18.0 发布：445个commit，213位贡献者**
vLLM 最新版带来 gRPC Serving 支持、NGram 推测解码 GPU 加速、KV Cache 智能卸载（FlexKV 后端）、弹性专家并行 NIXL-EP 集成、FlashInfer 0.6.6 更新等。新增 Eagle3 推测解码支持 Qwen3.5 和 Kimi K2.5 MLA。
🔗 [vLLM v0.18.0 Release](https://github.com/vllm-project/vllm/releases)

**3. TensorRT-LLM 预发布：DeepSeek V3.2 NVFP4 优化 + KV Cache/推测解码集成**
TRT-LLM 最新预发布改进了 DeepSeek V3.2 的 NVFP4 indexer GEMM 和路由内核，在 Trtllm-Gen attention 后端新增 KV cache 和推测解码支持。同时带来 FP8 combine MoE A2A、fused AllReduce+RMSNorm、Qwen3.5 初始支持。
🔗 [TensorRT-LLM Releases](https://github.com/NVIDIA/TensorRT-LLM/releases)

**4. llama.cpp b8571 持续迭代：JSON Schema 正则修复 + 多平台构建**
llama.cpp 在 3月28日密集发布多个构建版本，修复了 JSON schema pattern 中非捕获组 `(?:...)` 的 SIGSEGV 崩溃问题（影响 OpenAI 兼容 tool calls）。新增 CUDA 13.1、ROCm 7.2、OpenVINO 2026.0 等平台支持。
🔗 [llama.cpp b8571 Release](https://github.com/ggml-org/llama.cpp/releases)

**5. SGLang-Diffusion 进入生产级：token级序列分片 + FP8 + 并行VAE解码**
SGLang 扩展到图像生成推理领域，带来 token-level sequence sharding、parallel VAE decoding、fused kernels、Nunchaku 和 FP8 支持，以及 ComfyUI 插件的多个新模型支持。
🔗 [SGLang Diffusion Blog](https://lmsys.org/blog/)

## 🔧 技术速报

- **🧮 KV Cache 优化**：vLLM v0.18.0 引入智能 CPU 卸载（仅存储高频复用 block）+ FlexKV 新后端；TRT-LLM 新增 KVCacheManagerV2 的 Python 调度器和约束式内存分区
- **⚡ 推测解码**：vLLM 的 NGram speculative decoding 迁移到 GPU 并兼容 async scheduler；Eagle3 新增对 Qwen3.5 和 Kimi K2.5 MLA 的支持；TRT-LLM 在 Trtllm-Gen 后端集成推测解码
- **🔥 低精度推理**：SGLang 新增 FA4 (FP4 attention) 用于多模态编码器；TRT-LLM 改进 DeepSeek V3.2 NVFP4 内核；SGLang-Diffusion 支持 FP8
- **🏗️ MoE 并行**：vLLM 弹性专家并行 (NIXL-EP) 支持动态 GPU 扩缩容；SGLang FlashInfer All-to-All MoE Dispatcher 优化路由；TRT-LLM 新增 FP8 combine MoE A2A
- **🔗 Blackwell 平台**：SGLang v0.5.9 在 GB300 NVL72 上针对长上下文推理做 prefill-decode 分离优化

## 📈 今日洞察

本周三大推理框架密集更新，核心趋势非常明确：**MoE 模型的高效推理成为第一优先级**。SGLang 和 TRT-LLM 围绕 DeepSeek V3.2 的稀疏注意力内核展开深度合作，vLLM 则通过弹性专家并行 (NIXL-EP) 和 Eagle3 推测解码跟进。另一个显著方向是**精度下沉**——FP4 attention (FA4)、NVFP4 内核优化、FP8 KV Cache 已成为各框架标配，Blackwell GPU 的原生低精度支持正在加速这一趋势。值得关注的是 SGLang-Diffusion 的生产级优化，标志着推理加速框架正从纯文本扩展到图像生成领域。
