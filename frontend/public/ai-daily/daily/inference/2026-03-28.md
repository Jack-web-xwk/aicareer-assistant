## 📦 框架更新

### vLLM v0.18.0（2026-03-20 发布）
- 🌐 **gRPC Serving** — 新增 `--grpc` 标志，支持高性能 RPC 推理服务
- 🖥️ **GPU-less Render Serving** — 新命令 `vllm launch render`，支持无 GPU 预处理
- 🎯 **NGram GPU 推测解码** — 推测解码迁移到 GPU，兼容 async scheduler
- 📦 **KV Cache 卸载增强** — 智能 CPU 卸载只存高频复用 block + FlexKV 后端
- 🔄 **弹性专家并行 Milestone 2** — NIXL-EP 集成，支持动态 GPU 扩缩容
- ⚡ **FlashInfer 0.6.6** — 大量性能和正确性改进
- 🤖 **新模型架构** — Sarvam MoE、OLMo Hybrid、HyperCLOVAX、Kimi-Audio、ColPali 等
- 📝 **Responses API 流式工具调用** — OpenAI 兼容 API 支持 streaming tool calls
- 🔧 Ray 不再作为默认依赖

## 🔧 技术速报

- **Marlin 量化正确性** — vLLM 修复了 Marlin 量化内核在 float16 下的 NaN/Inf 问题，使用 GPTQ/AWQ 量化模型的用户应关注此修复
- **FlexKV 跨框架统一** — 同日 vLLM 和 TRT-LLM 均合入 FlexKV 相关代码，KV Cache 卸载正在成为行业标准能力
- **Qwen3.5 生态完善** — SGLang 修复 MoE Triton 调优 bug，vLLM 支持 Eagle3 推测解码 + FP8 KV Cache（B200 有已知精度问题）
- **MoE 推理加速** — vLLM 通过 torch.compile 融合 topk、弹性专家并行等手段持续优化 MoE 模型推理
- **推测解码演进** — vLLM v0.18.0 将 NGram 推测解码迁移至 GPU + async scheduler，显著降低 spec decode 开销
