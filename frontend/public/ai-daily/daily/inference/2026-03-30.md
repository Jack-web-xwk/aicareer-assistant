## 🔥 要闻

### 1️⃣ TRT-LLM 博客：MoE 单侧 AllToAll 通信优化（03/16）
NVIDIA 发布 TensorRT-LLM 技术博客第18篇，针对 MoE 模型在 NVLink 上的通信瓶颈，提出 One-Sided AllToAll 策略，减少专家并行中的同步开销。对 DeepSeek-V3 等超大规模 MoE 模型部署至关重要。
🔗 [NVIDIA TensorRT-LLM Tech Blog #18](https://github.com/NVIDIA/TensorRT-LLM/blob/main/docs/source/blogs/tech_blog/blog18_Optimizing_MoE_Communication_with_One_Sided_AlltoAll_Over_NVLink.md)

### 2️⃣ TRT-LLM Sparse Attention（03/04）
NVIDIA 发布 Sparse Attention 实现，优化长上下文推理的注意力计算效率，减少不必要的计算和内存消耗。
🔗 [Sparse Attention in TensorRT LLM](https://nvidia.github.io/TensorRT-LLM/blogs/tech_blog/blog17_Sparse_Attention_in_TensorRT-LLM.html)

### 3️⃣ TRT-LLM Skip Softmax Attention 加速长上下文推理（02/06）
通过跳过部分注意力 Softmax 计算，在长上下文场景下实现显著加速，进一步降低推理延迟。
🔗 [Skip Softmax Attention](https://nvidia.github.io/TensorRT-LLM/blogs/tech_blog/blog16_Accelerating_Long_Context_Inference_with_Skip_Softmax_Attention.html)

### 4️⃣ SGLang GB300 NVL72：25x 推理性能提升
SGLang 在 NVIDIA GB300 NVL72 上实现 25 倍推理性能提升，展示了新一代 Blackwell 架构在大规模推理部署上的巨大潜力。
🔗 [lmsys.org blog](https://lmsys.org/blog/2026-02-20-gb300-inferencex/)

### 5️⃣ llama.cpp b8571 持续高频迭代
llama.cpp 在 3月28-29日密集发布多个版本（b8571 及之前），修复 JSON Schema 正则表达式中非捕获组 `(?:...)` 的 SIGSEGV 崩溃，提升结构化输出稳定性。同时新增 CUDA 13.1 支持、ROCm 7.2 构建等。
🔗 [llama.cpp Releases](https://github.com/ggml-org/llama.cpp/releases)

## 🔧 技术速报

- 🧠 **量化支持现状**：SGLang 已支持 FP4/FP8/INT4/AWQ/GPTQ 全套量化方案，配合 Blackwell GPU 的 FP4 硬件加速能力，部署门槛持续降低
- ⚡ **vLLM Semantic Router v0.2**：新增模型路由、语义缓存、安全防护、内存管理和检索等多模块，从单纯推理引擎向完整推理基础设施演进
- 🔧 **TRT-LLM EP（专家并行）优化持续推进**：针对 DeepSeek-V3.2 等超大规模 MoE 模型，已在 GB200 上实现 3.8x Prefill / 4.8x Decode 吞吐提升
- 📱 **llama.cpp 硬件覆盖扩展**：新增华为 Ascend（310p/910b）、Intel OpenVINO 2026.0、SYCL 等构建版本，边缘推理生态进一步完善
