## 2. Harness 代码架构分层规范·强制约束

【架构逻辑·强制约束】采用五层分层架构，从上到下依次为：输入治理层→Prompt工程层→执行运行时→输出治理层→观测与管控层，层层递进，单向依赖，禁止反向依赖

【代码目录映射·强制约束】AI必须按照以下目录结构生成代码，每个层级对应一个独立的代码包，不得随意调整目录结构

### 2.1 输入治理层（input_governance）

**核心职责·强制约束**：处理用户原始请求，过滤脏输入、明确用户需求、保障输入合规性，是系统的前置防线，对应大模型调用链路的“Query预处理”

**核心代码模块·强制约束**：AI必须实现以下独立模块，每个模块对应一个独立的Python文件

- preprocessor.py：Query归一化预处理，包括无效字符过滤、乱码处理、文本纠错、格式统一

- safety_check.py：输入安全合规校验，包括敏感词检测、违规内容拦截、Prompt注入风险检测

- auth.py：请求鉴权与权限校验，包括API密钥校验、用户权限管控、接口限流

- intent_recognition.py：用户意图识别与请求路由，包括意图分类、场景匹配、请求分发

**统一接口规范·强制约束**：本层所有模块必须实现handle()方法，入参为Request对象，出参为ProcessResult对象，异常统一抛出InputHandleException

### 2.2 Prompt工程层（prompt_engine）

**核心职责·强制约束**：将预处理后的用户Query，转化为LLM可精准理解、可稳定执行的标准化Prompt，对应大模型调用链路的“make prompt”，决定LLM输出质量

**核心代码模块·强制约束**：AI必须实现以下独立模块，每个模块对应一个独立的Python文件

- context_manager.py：会话上下文管理，包括历史对话拉取、上下文压缩、滑动窗口截断、会话状态维护

- template_manager.py：Prompt模板管理，包括系统指令模板、任务模板、格式模板的加载、渲染、版本管理

- external_injector.py：外部信息注入，包括RAG知识库召回内容、工具调用结果、业务数据的格式化注入

- effect_enhancer.py：效果增强模块，包括Few-shot示例注入、CoT思维链引导、反思机制注入

- safety_reinforce.py：Prompt安全加固，包括防注入约束、合规规则注入、拒答规则定义

- prompt_builder.py：最终Prompt组装与校验，包括多模块内容拼接、Token数计算、长度管控、合规校验

**统一接口规范·强制约束**：本层所有模块必须实现build()方法，入参为SessionContext对象，出参为Prompt对象，异常统一抛出PromptBuildException

### 2.3 执行运行时（runtime）【核心重构层·强制约束】

**核心职责·强制约束**：Harness系统的核心中枢，管控LLM执行全流程，处理复杂任务流转，保障服务稳定，所有模型调用、工具调用、任务调度必须在此层完成，禁止在其他层直接调用LLM

**核心代码模块·强制约束**：AI必须实现以下独立模块，每个模块对应一个独立的Python文件

- model_scheduler.py：多模型调度与管理，包括模型接口统一封装、负载均衡、主备切换、模型参数配置

- llm_executor.py：LLM核心推理执行，包括API调用、流式处理、超时控制、异常捕获

- tool_executor.py：工具调用执行，包括函数定义解析、参数校验、工具执行、结果格式化、失败重试

- agent_loop.py：Agent核心循环，包括任务拆解、子步骤执行、状态管理、循环控制、中断恢复

- fault_tolerance.py：容错与降级处理，包括异常重试、熔断机制、降级策略、兜底处理

- state_manager.py：任务状态持久化，包括任务状态存储、断点续跑、会话状态同步

**统一接口规范·强制约束**：本层所有模块必须实现invoke()方法，入参为Prompt对象/Task对象，出参为LLMResult对象，异常统一抛出RuntimeException

### 2.4 输出治理层（output_governance）

**核心职责·强制约束**：处理LLM原始输出，校验内容合规性、准确性，适配业务需求，是系统的最后一道防线，对应大模型调用链路的“make output”

**核心代码模块·强制约束**：AI必须实现以下独立模块，每个模块对应一个独立的Python文件

- parser.py：输出内容解析与归一化，包括流式内容拼接、格式解析、乱码处理、内容归一化

- safety_audit.py：多层级合规安全审核，包括敏感内容检测、合规性校验、边界校验，违规内容100%拦截

- quality_check.py：内容质量校验，包括幻觉检测、事实性校验、逻辑一致性校验、完整性校验

- business_adapter.py：业务适配优化，包括业务规则校验、内容风格适配、个性化调整、敏感数据脱敏

- fallback_handler.py：异常兜底处理，包括异常场景判断、重试触发、兜底话术返回

**统一接口规范·强制约束**：本层所有模块必须实现process()方法，入参为LLMResult对象，出参为OutputResult对象，异常统一抛出OutputHandleException

### 2.5 观测与管控层（observability）

**核心职责·强制约束**：保障全链路可追溯、可监控、可迭代，所有模块必须接入本层能力，禁止出现无埋点、无日志的逻辑

**核心代码模块·强制约束**：AI必须实现以下独立模块，每个模块对应一个独立的Python文件

- trace_logger.py：全链路日志埋点，包括请求全流程日志、异常日志、审计日志，所有日志关联唯一TraceID

- metrics_collector.py：核心指标采集，包括耗时、Token用量、成功率、异常率、成本等核心指标采集

- alert_manager.py：告警管理，包括异常指标告警、阈值配置、告警渠道分发

- cost_manager.py：成本管控，包括Token用量统计、费用核算、配额管控、成本预警

- feedback_collector.py：用户反馈收集与迭代闭环，包括用户反馈存储、负向反馈复盘、策略优化支撑

**统一接口规范·强制约束**：本层所有模块必须实现record()方法，入参为TraceContext对象，无出参，异常不得影响主链路执行

## 3. 核心接口与代码实现强制约束

**3.1 统一数据对象规范·强制约束**：AI必须在common模块中定义以下统一数据类，全链路所有模块必须使用统一数据对象传递，禁止使用零散参数传递

- Request：用户原始请求对象，包含query、session_id、user_id、trace_id、request_time等核心字段

- SessionContext：会话上下文对象，包含预处理后的query、用户意图、会话历史、用户信息、外部注入数据等全链路上下文

- Prompt：最终送入LLM的Prompt对象，包含system_prompt、user_prompt、messages、token_count、model_config等核心字段

- LLMResult：LLM原始输出对象，包含raw_output、token_usage、inference_time、model_name、request_id等核心字段

- OutputResult：最终交付结果对象，包含final_output、status_code、error_msg、trace_id、process_time等核心字段

- TraceContext：全链路追踪对象，贯穿请求全流程，包含trace_id、各环节处理结果、耗时、异常信息、日志等核心字段

## 6. 常见问题与解决方案（AI开发参考）

- 问题1：Prompt注入/模型越权 → 解决方案：固定系统指令优先级，添加防注入约束，明确拒答规则，输入环节检测注入风险

- 问题2：LLM幻觉严重 → 解决方案：加入RAG知识库增强、输出环节事实性校验，注入Few-shot示例与CoT思维链引导

- 问题3：服务不稳定、易崩溃 → 解决方案：配置分级异常处理、重试、熔断、降级机制，所有异常有兜底，非核心异常不中断主链路

- 问题4：Token成本失控 → 解决方案：精细化Token管控，上下文滑动窗口截断，搭建高频请求缓存机制，按任务复杂度匹配对应模型

- 问题5：多轮对话上下文混乱 → 解决方案：标准化会话上下文管理，按优先级截断历史对话，长会话采用摘要压缩，固定核心指令不被截断

## 7. 核心术语对照表（AI开发统一认知，避免歧义）

- Harness：AI驾驭层，本次重构的核心主体，保障大模型工程化落地的全套管控体系

- LLM：Large Language Model（大语言模型），仅作为可插拔的核心推理组件，不包含任何工程化管控逻辑

- Token：大模型处理文本的基本单位，用于长度管控与计费
