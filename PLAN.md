# aicareer-assistant 产品化分析 — 执行计划

> 基于 PRODUCT_ANALYSIS.md 分析报告的落地计划

---

## 当前项目状态

- **定位：** AI求职工具（简历优化+面试模拟+职位搜索）
- **技术栈：** FastAPI + LangGraph + React 18 + SQLite
- **规模：** 后端约200KB，前端12个页面，测试覆盖率25%
- **阶段：** MVP可用，但缺生产级基础设施

---

## 核心问题（按优先级）

### P0 — 不解决这些，产品化无从谈起
1. **无用户认证** — 所有数据无归属
2. **SQLite存储** — 不支持并发，生产必挂
3. **内存会话存储** — 重启丢数据
4. **同步音频阻塞** — 单次问答延迟8.6秒

### P1 — 尽快解决
5. WebSocket连接泄漏
6. 爬虫反爬弱
7. 测试覆盖率25%
8. 前端大组件难维护
9. 简历解析能力有限

### P2 — 有余力再做
10. 无CI/CD
11. 无监控告警
12. 安全隐患（API Key硬编码、CORS过宽）

---

## 竞品核心差距

| 维度 | 本项目 | 领先竞品 |
|------|--------|---------|
| 用户系统 | ❌ 无 | ✅ 完整 |
| 数据库 | SQLite | PostgreSQL+Redis |
| ATS评分 | ❌ 无 | ✅ 成熟 |
| 移动端 | ❌ 仅Web | ✅ 多端 |

**核心优势：** 中文本地化、简历+面试组合（国内稀缺）、多LLM支持、LangGraph可观测性
